# 中国大学生数学建模竞赛项目规则

本文件适用于项目根目录及其全部子目录。除非用户明确提出更严格的要求，所有分析、代码、图表、论文与交稿文件都必须遵守以下规则。

## 1. 数据与事实完整性

- 不修改、覆盖、重命名或删除原始数据、赛题附件和原始题面。若项目没有 data/raw 目录，将首次收到的题面和附件视为只读原始材料。
- 需要清洗或转换数据时，将结果写入 data/processed 或项目既有的派生数据目录，并记录输入文件、处理脚本、参数、输出文件和必要的校验信息。
- 不伪造、猜测或补造数据、指标、实验结果、图表数值、参考文献及文献元数据。缺失信息必须明确标为未知、待确认或模型假设。
- 区分题目已知量、外部公开数据、模型假设、估计值和计算结果；在报告与论文中说明来源。

## 2. 可复现性

- 所有结论必须能从保存的原始材料、代码、参数与命令重新得到。不得把手工修改后的表格或图片作为唯一结果来源。
- 从项目根目录运行脚本，优先使用相对路径；记录关键软件版本、依赖、入口命令和运行顺序。
- 所有随机算法必须显式设置并记录随机种子。多次重复实验时记录种子集合、重复次数以及汇总方式。
- 中间结果和最终结果应写入明确的派生目录，例如 outputs、reports、paper/figures 或项目已有的对应目录；不得反向写入原始数据目录。
- 对运行失败、数据不足或无法复现的结论停止传播，不得在论文中把它表述为已验证事实。

## 3. 公式、代码、图表与论文一致性

- 维护统一的变量和符号表。公式、求解代码、数据字段、图表标签、表格、正文和附录中的符号定义、单位、下标、方向约定及数值必须一致。
- 报告数值时保留未舍入的机器结果，并在展示层统一舍入规则。百分数、角度、距离、时间、货币和无量纲量必须标明单位或定义。
- 模型参数或结果变更后，同步更新代码、派生数据、图表、表格、正文、摘要、结论和附录，随后重新执行验证与编译。
- 图表必须由代码生成，并保存生成脚本、输入数据和输出路径。禁止使用图片编辑器手工改变数据点、坐标、图例或结论性标注。
- 图表应具备中文论文所需的清晰标题、坐标、单位、图例和可辨识配色；用于论文的位图应有足够分辨率，能使用矢量格式时优先使用矢量格式。

## 4. 模型与验证

- 先建立可解释的基线模型，再比较候选模型并选择主模型。说明选择标准、约束条件、适用范围和失败情形。
- 任何主结论在写入论文前必须完成与任务相适应的误差、残差、敏感性、稳健性、基线比较或约束检查。
- 验证失败时，保留失败记录并返回模型开发阶段；不得通过删点、改口径或只展示有利实验掩盖问题。
- 训练、验证和测试数据必须按任务逻辑隔离；存在时间、空间、个体或组别相关性时避免信息泄漏。

## 5. 中文论文与 XeLaTeX

- 项目论文使用中文撰写，使用 XeLaTeX 编译。主文件优先查找 paper/main.tex；若不存在，查找包含 documentclass 和 document 环境、能够独立编译且最符合论文入口特征的主要 .tex 文件。
- 修改 .tex、参考文献、被引用图表或模板后必须重新编译，不得只修改源文件而沿用旧 PDF。
- 默认开启 SyncTeX，并使用可定位行号的编译参数。至少编译到交叉引用稳定；使用 BibTeX 或 Biber 时按项目实际工具链执行完整循环。
- 编译后检查 PDF 页数、目录、公式、图表、表格、中文字体、引用、超链接、页眉页脚和空白页，并确认 PDF 的时间戳与本次构建一致。

## 6. LaTeX 诊断分类

- 编译错误：包括非零退出码、LaTeX Error、Emergency stop、Fatal error、缺失文件、未闭合环境等。此类问题阻断交付，必须修复后重新编译。
- ChkTeX 警告：属于静态规范或风格检查结果，单独列出规则号、文件和行号；逐项判断，不得把它误报为编译失败。
- 普通排版警告：包括 Overfull/Underfull box、字体替代、浮动体位置、需要再次运行以更新引用等。它们不一定阻断构建，但必须检查其视觉影响；未定义引用和引用未稳定必须在交付前消除。
- 报告构建状态时分别汇总以上三类，不得只说“有警告”或“编译成功”而不说明 PDF 是否生成、引用是否稳定及排版是否检查。

## 7. 工作流与交付

- 按需调用项目 Skills：problem-analysis、data-audit、model-development、model-validation、paper-writing、latex-build、competition-paper-review、final-review。
- 在论文首次稳定编译后使用 competition-paper-review 模拟竞赛评委进行诊断性评审；其评分不冒充官方评分，发现的问题必须按证据退回对应 Skill 修正。修正后重新执行 latex-build 和 competition-paper-review，再进入 final-review。
- 不让后续阶段代替前置验证。论文写作只能使用已核验结果，最终审查只能确认或退回问题，不能凭空补齐模型证据。
- 保留用户已有有效内容和无关改动。修改已有文件前先读取；除非用户明确授权，不覆盖原始成果或删除历史结果。
- 交付时列出主要输入、代码入口、结果文件、论文源文件、最终 PDF、随机种子和仍存在的限制。
